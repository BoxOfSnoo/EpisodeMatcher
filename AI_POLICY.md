# AI Usage Policy

## Background

EpisodeMatcher was heavily developed through agentic coding — meaning a significant portion of the codebase was written or refined with the assistance of AI coding agents (such as GitHub Copilot and similar tools). This is part of the project's history and is acknowledged openly.

## Policy for Contributors

AI-assisted contributions are **explicitly welcome** in this project. You are free to use any AI tools — including large language models, code-completion assistants, and agentic coding workflows — when contributing code, documentation, or other assets.

### Requirements for AI-Assisted Submissions

The only firm requirement for AI-assisted contributions is **mandatory manual review and verification**:

1. **Read and understand every line you submit.** AI-generated code must be reviewed by a human before being included in a pull request. You are responsible for the correctness of your submission regardless of how it was produced.

2. **Test your changes manually.** Run the project locally, exercise the changed code paths, and confirm the output matches expectations. Do not submit code that you have not personally verified works.

3. **Do not blindly trust AI output.** AI tools can generate plausible-looking but incorrect, insecure, or inefficient code. Review for logic errors, security issues, and adherence to the project's architecture before submitting.

4. **Disclose significant AI involvement (optional but appreciated).** If your pull request was largely AI-generated, a brief note in the PR description is encouraged. This helps reviewers calibrate their review depth.

## Rationale

AI tools accelerate development and lower the barrier to contribution — both of which are good for this project. At the same time, unchecked AI output introduces risk. This policy strikes a balance: embrace AI tooling, but keep a human in the loop for all submissions.

---

*This policy applies to all contributions including code, documentation, and configuration changes.*
